# Windows PCでの動作確認手順（初心者向け）

このガイドでは、Windows PCでGroove Samplerアプリケーションを動作確認する方法を、プログラミング経験がない方にも分かりやすく説明します。

## 📋 必要なもの

1. **Node.js**（ノード・ジェイエス）というソフトウェア
   - これはアプリケーションを動かすために必要な「エンジン」のようなものです
   - 無料でダウンロードできます

2. **Google Chrome** または **Microsoft Edge** ブラウザ
   - MIDI機能を使う場合は、これらのブラウザが必要です

3. **コマンドプロンプト**（コマンド・プロンプト）
   - Windowsに最初から入っているツールです
   - 黒い画面に文字を入力して操作する画面です

## 🔧 用語の説明

### コマンドプロンプトとは？
- Windowsに標準で入っている「黒い画面のツール」です
- 文字のコマンド（命令）を入力して、パソコンに作業をさせます
- 例：フォルダを開く、ファイルをコピーする、など

### Node.jsとは？
- アプリケーションを動かすために必要な「ソフトウェア」です
- コマンドプロンプトから使います
- インストールすると、`node`や`npm`というコマンドが使えるようになります

**関係性**：コマンドプロンプト（黒い画面）を使って、Node.jsのコマンドを実行します

## 📥 ステップ1：Node.jsをインストールする

### 1-1. Node.jsがインストールされているか確認

1. **Windowsキー**を押して「cmd」と入力します
2. **コマンドプロンプト**をクリックして開きます
   - 黒い画面が開きます

3. 黒い画面に以下の文字を入力して、**Enterキー**を押します：

```
node --version
```

4. バージョン番号（例：v20.10.0）が表示されれば、**既にインストール済み**です
   - その場合は、次の「ステップ2」に進んでください

5. 「'node' は、内部コマンドまたは外部コマンド、操作可能なプログラムまたはバッチ ファイルとして認識されていません。」と表示されたら、**インストールが必要**です

### 1-2. Node.jsをインストールする（必要な場合）

1. ブラウザで以下のサイトを開きます：
   ```
   https://nodejs.org/
   ```

2. 画面に「LTS」と書かれた緑色のボタンが表示されます
   - LTS = 長期間サポートされる安定版です
   - このボタンをクリックします

3. ダウンロードが始まります
   - ファイル名は `node-v20.x.x-x64.msi` のような名前です

4. ダウンロードが完了したら、そのファイルをダブルクリックします

5. インストールウィザードが開きます：
   - 「Next」ボタンを何度かクリックします
   - 「Install」ボタンをクリックします
   - インストールが完了したら「Finish」をクリックします

6. **重要**：インストール後、**コマンドプロンプトを閉じて、再度開きます**
   - これで新しい設定が反映されます

7. 再度 `node --version` を実行して、バージョンが表示されることを確認します

## 📂 ステップ2：プロジェクトのフォルダを準備する

### 2-1. プロジェクトフォルダを開く

1. **エクスプローラー**（フォルダのアイコン）を開きます

2. Groove Samplerのプロジェクトフォルダを見つけます
   - 例：`C:\Users\あなたの名前\groove-sampler`
   - GitHubからダウンロードした場合は、そのフォルダです

3. そのフォルダの中身を確認します
   - `package.json` というファイルがあることを確認してください

### 2-2. コマンドプロンプトでフォルダを開く方法（簡単な方法）

1. エクスプローラーでプロジェクトフォルダを開いた状態にします

2. フォルダ内の何もないところで、**Shiftキーを押しながら右クリック**します

3. メニューから「**PowerShellウィンドウをここで開く**」または「**ターミナルウィンドウをここで開く**」を選択します
   - これで、そのフォルダでコマンドプロンプトが開きます

## 📦 ステップ3：必要なファイルをダウンロードする（依存関係のインストール）

アプリケーションを動かすには、追加のファイルが必要です。これらを自動的にダウンロードします。

1. コマンドプロンプト（黒い画面）で、以下の文字を入力します：

```
npm install
```

2. **Enterキー**を押します

3. ダウンロードが始まります
   - 「added xxx packages」のようなメッセージが表示されます
   - 数分かかる場合があります
   - 最後に「up to date」や「audited」というメッセージが表示されれば完了です

**注意**：エラーが出た場合は、インターネット接続を確認してください。

## 🚀 ステップ4：アプリケーションを起動する

1. コマンドプロンプトで、以下の文字を入力します：

```
npm run dev
```

2. **Enterキー**を押します

3. 以下のようなメッセージが表示されます：

```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
```

4. **この画面は閉じないでください**
   - 閉じるとアプリケーションが止まります
   - 最小化して、後ろに隠しておいても大丈夫です

## 🌐 ステップ5：ブラウザでアプリケーションを開く

1. **Google Chrome** または **Microsoft Edge** を開きます

2. アドレスバー（URLを入力する場所）に、以下を入力します：

```
http://localhost:5173
```

3. **Enterキー**を押します

4. Groove Samplerの画面が表示されます！

5. **最初の操作**：画面のどこかをクリックしてください
   - これで音が出るようになります（オーディオエンジンの初期化）

## ✅ 動作確認のポイント

- ✅ 画面が表示される
- ✅ パッドをクリックできる
- ✅ フェーダーを動かせる
- ✅ 音が出る（スピーカーやヘッドフォンが接続されている場合）

これで動作確認は完了です！

## 🎹 MIDI機能の使い方（オプション）

MIDIコントローラーをお持ちの場合は、物理的なノブやフェーダーでアプリケーションを操作できます。

### MIDIデバイスを接続する

1. MIDIコントローラーをUSBケーブルでPCに接続します

2. ブラウザでアプリケーションを開いた状態で、MIDIコントローラーを少し動かします

3. ブラウザの上部に「このサイトがMIDIデバイスへのアクセスを要求しています」というメッセージが表示されたら、**「許可」**をクリックします

### MIDI Learn機能の使い方（簡単3ステップ）

**MIDI Learn**とは、MIDIコントローラーのノブやフェーダーを、画面上のパラメータに割り当てる機能です。

1. **学習モードを開始**
   - 画面上のフェーダー（スライダー）の右上にある小さな **「M」ボタン**をクリックします
   - 「M」ボタンが**黄色で点滅**します
   - これで「学習モード」になりました

2. **MIDIコントローラーを動かす**
   - MIDIコントローラーの任意のノブやフェーダーを動かします
   - 自動的にマッピングが作成されます
   - 「M」ボタンが**青色**に変わります（マッピング完了の合図）

3. **完了！**
   - これで、そのMIDIコントロールを動かすと、画面上のパラメータが連動して動きます

### マッピングを削除する

マッピング済みのパラメータの「×」ボタンをクリックすると、MIDIマッピングを削除できます。

## ❓ よくある問題と解決方法

### 問題1：コマンドプロンプトに「'node' は認識されていません」と表示される

**原因**：Node.jsがインストールされていない、または設定が反映されていない

**解決方法**：
1. Node.jsをインストールします（ステップ1を参照）
2. インストール後、**コマンドプロンプトを完全に閉じて、再度開きます**
3. 再度 `node --version` を試してください

### 問題2：`npm install`でエラーが出る

**原因**：インターネット接続の問題、またはフォルダの場所が間違っている

**解決方法**：
1. インターネット接続を確認してください
2. プロジェクトフォルダ（`package.json`があるフォルダ）でコマンドを実行しているか確認してください
3. ウイルス対策ソフトがブロックしていないか確認してください

### 問題3：`npm run dev`を実行しても何も表示されない

**原因**：ポート番号が違う、またはエラーが発生している

**解決方法**：
1. コマンドプロンプトのメッセージをよく見てください
2. `http://localhost:5173` ではなく、`http://localhost:5174` など別の番号が表示されている場合があります
3. その場合は、表示されているURLをブラウザに入力してください

### 問題4：MIDIデバイスが認識されない

**原因**：ブラウザの設定やデバイスの接続の問題

**解決方法**：
1. **ブラウザの確認**：ChromeまたはEdgeを使用してください（Firefoxは使えません）
2. **デバイスの確認**：
   - Windowsキー + Xキーを押します
   - 「デバイスマネージャー」を選択します
   - 「サウンド、ビデオ、ゲームコントローラー」の下にMIDIデバイスが表示されているか確認します
3. **ブラウザの再起動**：ブラウザを完全に閉じて、再度開いてみてください
4. **MIDIアクセスの許可**：ブラウザの設定でMIDIアクセスがブロックされていないか確認します

### 問題5：音が出ない

**原因**：オーディオ設定の問題

**解決方法**：
1. **画面をクリック**：アプリケーションを開いた後、画面のどこかをクリックしてください（これで音が出るようになります）
2. **ブラウザの音量**：ブラウザのタブに音量アイコンがある場合、ミュートになっていないか確認してください
3. **Windowsの音量**：画面右下のスピーカーアイコンで、音量が0になっていないか確認してください
4. **スピーカー/ヘッドフォン**：接続されているか確認してください

## 📝 まとめ：重要なコマンド

動作確認に必要な主なコマンドは以下の3つです：

1. **`npm install`** - 必要なファイルをダウンロードする（最初の1回だけ）
2. **`npm run dev`** - アプリケーションを起動する（毎回必要）
3. **ブラウザで `http://localhost:5173` を開く** - アプリケーションを見る

## 💡 補足情報（上級者向け）

### アプリケーションを停止する

コマンドプロンプトで `Ctrl + C` キーを押すと、アプリケーションを停止できます。

### 本番用にビルドする（オプション）

他の人に配布するためのファイルを作る場合は：

```
npm run build
```

これで `dist` フォルダにファイルが作成されます。

## 🆘 それでも解決しない場合

1. エラーメッセージをコピーして、詳しい人に相談してください
2. コマンドプロンプトの画面をスクリーンショットして保存しておくと便利です
3. Node.jsのバージョンが古い可能性があるので、最新版に更新してみてください

**最後に**：最初は難しく感じるかもしれませんが、一度セットアップできれば、次回からは簡単です。頑張ってください！
